# P1 — Child Experience (`/child`)

**Mission:** the screen judges stare at. Avatar tap → audio-guided adaptive
assessment → speaking handoff → celebration. Mobile-first, zero reading required.

**You own:** `src/app/child/**`, `src/components/child/**`
**You consume:** `Item`, `SessionResult` from `types.ts`; `estimate`/`nextItem`/`levelBand`
from `src/lib/cat.ts` (P2); `<SpeakingSection>` (P4); `syncResult()` from
`src/lib/resultSync.ts` (P3). Mock all of them until they exist (`// SWAP:`).

## Agent split

- **Claude Code (main):** items 1–5 below, in order.
- **Codex (parallel, fire immediately):** `<JourneyBar>` + `<CelebrationScreen>`
  from the specs below, as standalone components with props only — no app wiring.

## Checklist (time-boxed)

### 0:10–0:25 — Shell + audio unlock
- [ ] `/child` page: full-screen avatar button (≥ 200 px, emoji avatar, bounce on tap)
- [ ] Tap handler: create ONE shared `AudioContext` + play a silent buffer
      (this unlocks mobile audio for the whole session — nothing plays before it),
      then start session
- [ ] `useAudio()` helper: `play(url): Promise<void>` + `replay()`, decoding via
      the shared context; preload next item's audio while current plays

### 0:25–0:55 — Item renderer, all 3 formats
- [ ] `<ItemScreen item onAnswer(choiceId, ms)>` with exactly 3 layouts keyed on
      `item.format`:
  - `hear-word` / `hear-sentence`: audio autoplays (post-unlock), 2×2 grid of
    choice cards (emoji ~80 px, rounded-3xl, thick border)
  - `see-word`: `item.prompt` huge (text-6xl) top, picture choices below
- [ ] Persistent replay button (🔊, bottom corner, ≥ 64 px)
- [ ] On tap: card pulses, ~800 ms warm-neutral response (rotate 6 canned audio-free
      phrases + star burst — recorded audio can swap in later), then advance
- [ ] Record `{ itemId, choiceId, correct, ms }` per answer into local state
      AND localStorage (`session-in-progress`) so a killed tab can resume

### 0:55–1:15 — Adaptive wiring + flow
- [ ] Session loop: pool from `src/lib/itemBank.ts` (mock until P2 lands), call
      `nextItem()` each step, stop after 15 tap items
- [ ] `<JourneyBar>` (Codex): mascot dot advancing across 18 stops; plus a small
      "level dots" indicator that visibly rises/falls with difficulty — judges
      must SEE adaptation
- [ ] After item 15 → render P4's `<SpeakingSection>` (mock: button that resolves
      a canned `speaking` object); always skippable
- [ ] Build `SessionResult` (θ via `estimate()`, band via `levelBand()`), call
      `syncResult(result)` (mock: console.log), → celebration

### 1:15–1:30 — Celebration + polish
- [ ] `<CelebrationScreen>` (Codex): confetti (CSS only), big badge, mascot,
      "You did it!" — NO numbers, no scores
- [ ] Run the full flow on your phone via the Vercel URL; fix only what breaks

## Definition of done
Full session (avatar → 15 items → speaking mock → celebration) completes on a
phone on the deployed URL, audio plays, difficulty indicator visibly moves, and
airplane mode after first load does not break it (P3's SW handles caching — your
job is just: no fetches inside the item loop).

## Do NOT
Build settings, login, item formats beyond the 3, or touch `cat.ts` — if the
engine seems wrong, tell P2.
