### A2 (0:10–0:25) — Shell + audio unlock

- [ ] `src/app/child/page.tsx` (`"use client"`): full-screen layout, one avatar
      button ≥ 200 px (emoji avatar, bounce on tap). Nothing else on screen.
- [ ] Tap handler: create the app's ONE shared `AudioContext`, play a silent
      buffer through it (this unlocks mobile audio for the whole session —
      nothing may play before this gesture), then start the session.
- [ ] `src/components/child/useAudio.ts`: module-level shared context +
      decoded-buffer cache (`Map<string, AudioBuffer>`).
      API: `play(url): Promise<void>` (fetch → decode → play, resolves on end),
      `replay()` (replays last url), `preload(url)` (decode into cache).
      Preload the NEXT item's audio while the current one plays.
- [ ] **Fail-soft audio rule:** `mockItems` have NO `audioUrl`. Missing url,
      failed fetch, or failed decode ⇒ resolve immediately + `console.log` —
      never a visible error, never a blocked screen (RULES §6).

### A4 (0:55–1:15) — Session state machine + adaptive wiring

Session phases (in `page.tsx` or `src/components/child/Session.tsx`):

```
intro → item(prompt-playing → awaiting-tap → feedback) ×15 → speaking → celebration
```

- [ ] Loop: pool from `itemPool` (mock until P2), `nextItem(pool, responses)` each
