# P2 — CAT Engine + Item Bank

**Mission:** the brain (adaptive engine) and the content (real items + audio).
Nobody sees your work directly; everybody's work runs on it.

**You own:** `src/lib/cat.ts`, `src/lib/cat.test.ts`, `src/lib/itemBank.ts`,
`scripts/**`, `public/audio/**`
**You provide:** the exact API below — P1 imports it, P3's server-side recompute
reuses `estimate()`.

## The frozen API (P1 builds against this from minute 10)

```ts
// src/lib/cat.ts
export function estimate(
  pool: Item[],
  responses: { itemId: string; correct: boolean }[]
): { theta: number; standardError: number };

export function nextItem(
  pool: Item[],
  responses: { itemId: string; correct: boolean }[]
): Item | null;   // null when pool exhausted; excludes already-answered items

export function levelBand(theta: number):
  "Emerging" | "Beginning" | "Developing" | "On Track";
// thresholds: < -1.5 Emerging, < -0.5 Beginning, < 0.5 Developing, else On Track
```

## Agent split

- **Claude Code (main):** the engine (below). Give it this exact spec:
  2PL model `P(correct) = 1 / (1 + exp(-a*(θ-b)))`; EAP estimation over a
  quadrature grid θ ∈ [-4, 4] step 0.1 with N(0,1) prior; `standardError` =
  posterior SD; `nextItem` = max Fisher information `a²·P·(1-P)` at current θ̂.
- **Codex (parallel, fire immediately):** `scripts/generate-items.ts` +
  `scripts/render-audio.ts` per specs below. The two halves never touch.
